# 高三资料领取招生系统说明书

## 以后怎么保存版本

以后代码保存、提交、上传 GitHub，默认由 Codex 处理。

你不用记终端命令。每次改完一版，只要说：“保存一版”，Codex 就负责检查当前改动、保存到 Git，并上传到 GitHub。

只有 GitHub 要输入密码或 token 时，需要你自己输入，因为账号密码不能交给别人。

## 系统定位

这是一个面向教培招生的最小运营单元原型：用一个资料领取 H5 页面，把家长从“看到活动”引导到“填写信息、领取资料、加老师微信、进群、后台跟进”。

后台记录客户来源、页面停留、点击次数、分享关系和跟进状态，帮助老师判断谁更值得优先联系。

## 家长端流程

1. 打开活动链接，查看标题、人数、截止时间和资料预览。
2. 点击“立即领取资料”，填写学生信息和学习问题。
3. 提交后看到领取成功页，复制老师微信或进群口令。
4. 可以把活动分享给同学或家长，系统记录分享来源。

## 后台模块

1. 活动管理：修改标题、副标题、名额、截止时间、老师微信、进群口令和分享文案。
2. 数据总览：查看访问、领取、诊断、试听、互动热度和今日跟进提醒。
3. 客户 CRM：查看客户信息、来源、意向分、页面行为、分享贡献和跟进状态。
4. 渠道链接：创建不同传播渠道链接。
5. 分享关系：查看谁分享后带来了哪些客户。
6. 资料内容：添加文字、图片、视频、语音、PDF 等资料内容。
7. 表单配置：设置家长需要填写的信息。
8. 公开设置：控制前台人数、近期动态、头像、脱敏和名单展示规则。
9. 使用说明：查看系统说明、已实现功能和待实现清单。

## 还没有实现

- 真实微信头像需要接微信授权，目前是模拟头像。
- 文件上传还没有接服务器存储，目前是本地原型数据。
- 后台账号登录、数据库、小程序版本还没有接入。

## 建议下一步

先补真正上线前必须接的基础能力：

1. 接服务器存储和数据库，避免刷新或换设备后数据丢失。
2. 加后台账号登录，区分老师、运营和管理员权限。
3. 接微信授权或小程序能力，获取真实头像和更稳定的分享卡片。
